# Pertemuan 2
Pertemuan 2 berisi pengantar. Kita akan diajak untuk mengenal data, memahami database, membandingkan DBMS, serta menyingkap potensi MariaDB. 
## Modul 1
Kita diperintahkan untuk mempelajari definisi data, perbedaan data dan informasi, serta contoh nyata sehari-hari. Modul pengantar ini menyiapkan fondasi sebelum memahami struktur tabel dan SQL.
## Modul 2
Modul ini membahas tentang data. Data adalah representasi fakta, angka, atau simbol yang disimpan dalam bentuk tertentu sehingga dapat diproses menjadi informasi (Silberschatz, Korth, & Sudarshan, 2020). Dalam konteks sistem informasi, data menjadi pondasi yang memungkinkan organisasi melakukan pencatatan dan analisis. 

Selain pengertian data, modul ini juga membahas tentang perbedaan data dan informasi, jenis-jenis data, representasi data dalam kehidupan sehari-hari, kualitas data, pentingnya pengelolaan data, dan contoh kasus perpustakaan.
## Modul 3
Modul ini membahas tentang database. Kita diajak untuk mengenali alasan mengapa database dibutuhkan untuk data dengan jumlah yang besar. Kita juga membahas Keterbatasan penyimpanan manual, konsistensi dan integritas data, efisiensi dan penghematan sumber daya, skalabilitas dan adaptabilitas, dan juga dampak database pada perpustakaan.
## Modul 4
Di modul ini, kita mempelajar tentang perbedaan mendasar DBMS dan file biasa. Modul ini membahas konsep database management system (dbms), perbedaan operasional file vs dbms, keamanan dan pemeliharaan, juga studi kasus perpustakaan. 
## Modul 5
Modul ini mengajak kita untuk mengenali MariaDB sebagai pemula. MariaDB lahir sebagai turunan dari MySQL setelah akuisisi MySQL oleh Oracle (Widenius, 2013). Di sini dibahas tentang fitur utama MariaDB, keunggulan MariaDB bagi organisasi, peran MariaDB dalam perpustakaan, dan studi kasus perpustakaan digital. 
## Modul 6
Modul ini membahas tentang potensi dari penggunaan database.

# Pertemuan 3
Pertemuan 3 mengajak kita untuk memahami struktur database, mengenal tabel, baris, kolom, serta praktik membuatnya.
## Modul 1
Membahas konsep database dan tabel. Di sini dibahas tentang definisi database, baris sebagai record data, kolom sebagai atribut data, relasi antar tabel, kolaborasi database, tabel, baris, dan kolom. 
## Modul 2
Modul ini membahas tentang tipe data sederhana. Di sini dijelaskan tentang definisi tipe data, tipe data teks (string), pentingnya pemilihan tipe data, dan studi kasus perpustakaan.
## Modul 3
Di modul ketiga ini kita akan belajar membuat database baru. Di sini dijelaskan mulai dari Persiapan, perintah dasar create database, verifikasi database yang dibuat, kesalahan umum, best practice, dan juga studi kasus perpustakaan.
## Modul 4
Modul ini mengajarkan kita untuk membuat tabel sederhana. Kita diajarkan mulai dari persiapan database, membuat tabel anggota, membuat tabel buku. Untuk itu kita juga diajari untuk pemilihan tipe data yang tepat, penggunaan constraint NOT NULL dan UNIQUE, juga dokumentasi struktur tabel.

# Pertemuan 4
Pertemuan 4 mengajak kita untuk menambahkan, menampilkan, mengubah, dan menghapus data, serta praktik mengisi tabel. 
## Modul 1
Modul ini membahas tentang persiapan tabel, perintah dasar INSERT, menambahkan beberapa data sekaligus. terdapat beberapa kesalahan umum, yaitu jumlah nilai tidak sesuai kolom, tidak menyebutkan semua kolom yang dibutuhkan, format data tidak tepat, dan ISBN duplikat. untuk mencegahnya, gunakan kolom yang spesifik, gunakan multi-row insert untuk efisiensi, perhatikan integritas data, dan validasi data sebelum insert.
## Modul 2
Modul ini mempelajari perintah SELECT * untuk melihat semua isi tabel. Modul ini menekankan pentingnya menampilkan data sebagai cara memverifikasi hasil INSERT. 
## Modul 3
Modul ini mengajarkan penggunaan perintah UPDATE untuk mengubah isi data pada tabel. Kita akan berlatih memperbarui informasi buku dengan kondisi tertentu.
## Modul 4
Kita akan mempelajari cara menghapus data dari tabel menggunakan perintah DELETE. Modul ini membantu memahami bagaimana menjaga kebersihan data dengan menghapus entri yang tidak diperlukan.
